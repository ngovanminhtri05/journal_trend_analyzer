Journal Trend Analyzer

Mô tả cấu trúc source code · Flutter mobile app (PRM393 Lab2)

Nguồn dữ liệu: OpenAlex API · State: Provider · 29 file Dart, ~1.826 dòng · 34 test

1. Tổng quan kiến trúc

Ứng dụng theo kiến trúc phân lớp (layered / clean separation), dữ liệu chảy một chiều. Tầng giao diện không gọi API trực tiếp — chỉ đọc trạng thái từ Provider; Provider gọi Service; Service nói chuyện với OpenAlex và parse JSON. Mỗi lớp một trách nhiệm nên dễ test và bảo trì.

OpenAlex API→ services/
HTTP + parse + lỗi→ state/
Provider + ViewState→ screens/ + widgets/
UI

models/ (khuôn dữ liệu) dùng chung cho services và UI. theme/ định nghĩa giao diện tập trung.

6. Kỹ thuật & chất lượng

  • Async: Future/async-await, Future.wait tải song song, timeout 15s.
  • Xử lý lỗi: exception có kiểu (Network / RateLimit 429 / Parse) → UI hiện message + Retry.
  • OpenAlex aggregation: dùng group_by để server thống kê sẵn (nhẹ client).
  • Tách lớp & test: http.Client được inject → test bằng MockClient không cần mạng.
  • Design system: editorial minimalism tập trung ở app_theme.dart (google_fonts).
  • Kiểm thử: 34 test (model, decode abstract, URL/parse/lỗi service, chuyển trạng thái provider, luồng search) · flutter analyze: 0 lỗi
